# oh-my-projectskill 视频介绍脚本

---

**【开场 - 痛点引入】**

大家好，今天介绍一个项目：oh-my-projectskill。

做开发的人应该都有过这样的体验——clone 一个新项目，打开 README，看到一长串安装步骤。装依赖、配环境变量、选运行时、认证服务……每一步都可能踩坑。

传统方案要么写文档，用户得自己一步步执行；要么写 shell 脚本，但遇到需要用户选择的场景就不行了。

**【核心概念】**

oh-my-projectskill 的思路很简单：把安装文档变成 Agent Skill，让 AI 来执行。

它是一种项目级别的 Skill 模板，写成 Markdown 格式，Claude Code 可以直接理解和执行。跟传统脚本比，它多了四个关键能力：

- 环境检测——自动判断你的系统平台和已有工具
- 交互决策——需要你做选择时，Agent 会暂停来问你
- 条件分支——根据检测结果走不同的安装路径
- 验证反馈——每步完成后自动验证，失败了还能自动排查

**【实际案例】**

这个项目的灵感来自 NanoClaw——一个基于 WhatsApp 的 AI Agent 平台。部署它需要 9 个步骤，几乎每步都涉及环境判断和用户选择。

有了 ProjectSkill，用户只需要说一句"帮我安装配置这个项目"，Agent 就会自动跑完所有流程，该问你的时候问你，不该打扰的时候静默执行。

**【扩展场景】**

除了 setup，这个思路还能用在更多场景：项目健康检查、部署流程、数据库迁移、新人 onboarding、发版流程——都可以写成 Skill。

**【收尾】**

一句话总结：ProjectSkill 让项目文档活起来，环境配置不再是入门障碍，而是一次和 AI 的自然对话。

项目地址在 GitHub https://github.com/tobegit3hub/oh-my-projectskill，欢迎大家试用。
